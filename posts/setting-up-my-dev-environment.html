Setting Up My Dev Environment

06/06/2026

GitHub, VS Code, and Godot

This week's blog post is focused on Module 0, Setup and Tooling. In this module, I downloaded Visual Studio Code and set up my GitHub. Part of that was learning how to create repositories, make commits, and push those commits to GitHub. I followed GitHub's intro tutorial and made my “Hello World” repository. Getting everything set up wasn't too difficult, but it was time-consuming.

The next part of this module was downloading Godot. Fortunately, I had already been messing around with Godot, so that part was already done. Overall, I had most of this done already. The toughest part of setting everything up was GitHub Pages and Jekyll. I would use that to set up and start this blog. (I've since moved the blog to this portfolio site.)

Getting this blog set up was, I think, the most difficult thing I had done. It took banging my head against the wall for much longer than I wanted (or needed) to, but most of that was because I was stubborn. I spent hours trying to figure out how to get everything set up correctly, jumping between the GitHub Pages tutorial and Jekyll documentation to figure out why pages weren't looking how they should. It didn't help that I didn't really know much about HTML or Jekyll to really even know what I needed to look for. In the end, it turned out that all I had to do was follow the tutorial on the Jekyll website, rather than information I found on reddit. Always trust the official documentation over everything else. Lesson learned.
